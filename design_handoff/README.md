# 비트키퍼 (BeatKeeper) — 디자인 핸드오프 패키지

무대 위 실시간 템포 보조 앱의 **확정 디자인 시안 + 디자인 시스템**입니다.
이 패키지만으로 디자인을 **픽셀 단위로 재현·구현**할 수 있습니다.

---

## 1. 실행 방법

별도 빌드 없이 정적 파일입니다. 폴더에서 로컬 서버를 띄워 여세요(CDN·상대경로 때문에 `file://` 직접 열기는 비권장):

```bash
# 아무 정적 서버나 OK
npx serve .
# 또는
python3 -m http.server 8080
```

브라우저에서 `비트키퍼 연주화면.html` 열기 → 디자인 캔버스가 뜨고, 모든 화면이 **실시간 박동·인터랙티브**로 동작합니다.
(React 18 + Babel standalone을 CDN으로 로드하므로 첫 로드시 인터넷 필요. 컴포넌트/스타일은 전부 로컬 파일.)

---

## 2. 파일 구조

```
비트키퍼 연주화면.html   ← 메인. 디자인 토큰(CSS 변수)·전체 컴포넌트 CSS·캔버스 조립
manifest.json            ← PWA 매니페스트 (standalone · 다크 · orientation any)
assets/icons/            ← 실제 아이콘 에셋 (아래 §5)

── 핵심 엔진 ──
bk-core.jsx        메트로놈 클럭·박동 엔벨로프·드리프트 존 색상·BPM 시뮬레이션
bk-inst-core.jsx   인스트루먼트 상태 훅(useInstrument) + 캔버스 헬퍼
bk-draw.jsx        캔버스 드로잉 프리미티브(바늘 트랙·박자 점)

── 화면 ──
bk-inst-a.jsx      펄스 링(순환) 인스트루먼트 — 연주 화면 중앙
bk-screen.jsx      연주 화면 셸(헤더·컨트롤·하단탭) + 공연 모드 전환
bk-stage.jsx       공연 모드(세로 전체화면) + 가로 레이아웃
bk-ui.jsx          공유 UI 프리미티브(탑바·토글·세그먼트·스테퍼·슬라이더·리스트·하단탭)
bk-screens2.jsx    셋리스트 · 곡 편집 · 설정 화면
bk-extras.jsx      펄스 링 상태 변형 카드 · 앱 아이콘 렌더러

── 문서 ──
bk-specs.jsx       애니메이션 사양 · 디자인 토큰표 · 컴포넌트 시트
design-canvas.jsx  시안 비교용 캔버스 래퍼(제품 코드 아님 — 발표용 스캐폴드)
```

> 구현 시 `design-canvas.jsx`는 불필요합니다(시안 배치용). 실제 앱은 `bk-*` 컴포넌트와
> 메인 HTML의 CSS 토큰을 기준으로 옮기면 됩니다.

## 3. 디자인 토큰 (메인 HTML `:root`)

```
배경    #0a0b0e   배경2 #0e1016   표면 #15171e   표면2 #1c1f28   경계 #282b35
텍스트  #e9eaee   보조 #9aa0aa   흐림 #6b7280
강조    #38bdf8 (cyan)   강조잉크 #04141f   강조연함 rgba(56,189,248,.14)
드리프트 존  초록 #4fd1a5 · 노랑 #f4c95f · 빨강 #f2867f   (저자극용 소프트 톤)
모서리 14px · 최소 탭 타깃 52px
폰트:  Noto Sans KR (한글/본문) · IBM Plex Mono (숫자/스펙) — 둘 다 Google Fonts
```

판독 위계 (무대에서 0.5초 흘끗):
1. **링 색** = 드리프트 존 (초록 |Δ|≤1.5 · 노랑 ≤4 · 빨강 >4 BPM, 설정 가능)
2. **바늘 위치** = 앞섬/처짐 (왼쪽=느림/처짐, 오른쪽=빠름/앞섬, 풀스케일 ±8 BPM)
3. **큰 BPM 숫자** + 목표·Δ

## 4. 펄스 링 박동 — 애니메이션 사양 (Canvas + requestAnimationFrame)

| 항목 | 값 | 비고 |
|---|---|---|
| 주기 | `60000 / BPM` ms | 120 BPM → 500ms |
| 어택 | 주기의 6% | ≈30ms, 선형 상승 |
| 감쇠 | ease-out `(1−p)^1.7` | 나머지 94% 구간 |
| 크기 변화 | +5% (일반) / +6.5% (1박 강세) | 반경 scale |
| 글로우 blur | 8 → 34 px | env 비례 (shadowBlur) |
| 선 두께 | 4 → 9 px | env 비례 |
| 바늘 이징 | EMA `α = 0.08 / frame` | 떨림 억제 |
| 색 전환 | 즉시(존 경계) | 형태(바늘)+색 이중 표현 |
| 동기 | **오디오 클릭과 동일 타임스탬프** | 시각=청각 일치 (setInterval 드리프트 금지) |

구현 코드 기준: `bk-core.js` 의 `beatState()` / `bk-inst-a.jsx` 의 ring draw 루프.

> 참고: 이 프로토타입의 감지 BPM은 **데모용 시뮬레이션**(`simDetected`)입니다.
> 실제 마이크 템포 감지(스펙트럴 플럭스 온셋 → 자기상관)는 빌드 브리프 §6 참조.

## 5. 아이콘 에셋 (assets/icons/)

| 파일 | 용도 |
|---|---|
| `icon-192.png` | PWA 192 |
| `icon-512.png` | PWA 512 |
| `icon-maskable-512.png` | maskable (안전영역 78% 내 컨텐츠) |
| `apple-touch-180.png` | iOS 홈 화면 |
| `favicon.svg` | 파비콘(벡터) |

모티프 = **파동**(중심 발광점에서 퍼지는 시안 동심원) — 펄스 링 컨셉과 일치.
`manifest.json` 의 `icons`에 연결되어 있고, 메인 HTML `<head>`에 favicon/apple-touch/theme-color 링크됨.

## 6. 화면 목록

- **연주 화면 (세로)** — 일반 / 마이크 권한 필요
- **공연 모드 (세로 전체화면)** — 탭하면 컨트롤 등장
- **가로 (landscape)** — 일반 / 공연 (BPM은 링 안쪽 중앙)
- **셋리스트** · **곡 편집**(탭 템포 실작동·구간) · **설정**
- **상태 변형 6종** — 측정중 / 초록 / 노랑 / 빨강 / 신호없음 / 권한필요
- 디자인 토큰표 · 컴포넌트 시트 · 애니메이션 사양

*정답은 "무대에서 0.5초 안에 읽히는가". 예쁨보다 판독성·반응 속도·안정감 우선.*
