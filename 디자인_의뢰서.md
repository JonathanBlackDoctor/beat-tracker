# 비트키퍼(BeatKeeper) — 디자인 의뢰서 / Design Brief for Designer

> 디자이너에게 그대로 전달하는 문서입니다.
> 이 앱은 **이미 작동하는 시제품**이 있습니다. 새로 기획하는 게 아니라, 동작하는 앱의 **비주얼·UX를 전문가 수준으로 다듬는** 리디자인 작업입니다.

---

## 0. 한 줄 요약

밴드에서 카혼/드럼을 치는 연주자가 **무대 위에서** 폰을 흘끗 보고 "지금 빨라지나 느려지나"를 0.5초 안에 판단하는 **실시간 템포 보조 앱**(안드로이드 PWA, 다크 테마, 오프라인).

---

## 1. 무엇을 의뢰하나 (스코프)

- 4개 화면의 **비주얼 디자인 + 인터랙션 정리**: ① 연주 화면(핵심) ② 셋리스트 ③ 곡 편집 ④ 설정
- **핵심 컴포넌트** 디자인: 중앙 "펄스 링"(메트로놈 박자에 맞춰 깜빡이는 큰 원) + 앞섬/처짐 바늘 + 드리프트 색 상태(초록/노랑/빨강)
- 다크 테마 **디자인 시스템**: 색·타이포·간격·버튼/토글/입력 등 컴포넌트, 아이콘
- **앱 아이콘 / 스플래시** (PWA 설치용)
- 가능하면 **앱 이름·로고**도 제안 (현재 작업명 "비트키퍼", 미확정)

### 의뢰하지 않는 것
- 코드 구현(이미 되어 있음), iOS 전용 디자인(안드로이드만), 마케팅/스토어 페이지, 다국어(한국어만)

---

## 2. 사용 맥락 (디자인 판단의 기준)

이게 가장 중요합니다. 일반 앱과 전제가 다릅니다.

- **무대 위, 합주 중**에 본다 → 팔 거리(약 50~70cm)에서 **0.5초 흘끗** 보고 판단.
- 폰은 거치대에 **세로 또는 가로**로 고정 → 양쪽 방향 모두 디자인 필요.
- 어두운 무대 + 밝은 조명 변화 → **고대비 다크 테마** 필수, 빨강/노랑/초록이 한눈에 구분돼야 함.
- 연주 중 손이 바쁨 → 탭 타깃 크게(최소 52px), 정밀 조작 최소화.
- "공연 모드": 컨트롤 다 숨기고 펄스 링·BPM 숫자만 **화면 가득** 키우는 모드 있음.

## 5. 화면별 현재 상태와 요청

### ① 연주 화면 (핵심 — 80%의 노력 투입)
정보 위계(흘끗 판독 순):
1. **펄스 링** — 중앙 큰 원, 박마다 커졌다 작아짐. **링 색 = 드리프트 상태** (초록 `|Δ|≤1.5`, 노랑 `≤4`, 빨강 `>4` BPM).
2. **앞섬/처짐 바늘** — 링에 통합. 중앙=일치, 왼쪽=느림(처짐), 오른쪽=빠름(앞섬). 풀스케일 ±8 BPM.
3. **감지 BPM 숫자(대형)** + 목표 표기 (예: `124 → 목표 120, Δ +4`).
4. 안정도 미터(작은 막대), 박자 점(4/4면 점 4개·1박 강세), 컨트롤(시작/정지·곡 이동·다음 구간).
- **요청:** 1·2·3번이 무대에서 압도적으로 읽히는 위계. 색맹 고려한 색+형태 이중 표현 환영. 공연 모드(전체화면) 별도 레이아웃.

### ② 셋리스트 화면
- 여러 셋리스트, 곡 목록(곡명·BPM·박자표), 드래그 순서변경, 추가/편집/삭제/복제.
- **요청:** 공연 중에도 곡을 빠르게 찾아 누를 수 있는 리스트.

### ③ 곡 편집 화면
- 곡명, 목표 BPM, 박자표(4/4·3/4·6/8), 메모, 구간(섹션) 목록.
- BPM 입력 3가지: 숫자 스테퍼/키패드, **탭 템포**(연타), **자동 인식**.
- **요청:** 입력 폼이지만 답답하지 않게.

### ④ 설정 화면
- 클릭음 종류·볼륨, 1박 강세, 카운트인, 잘게 쪼개기, 드리프트 색 임계값, 진동, 측정 기본 모드, 화면 항상 켜둠, 언어.
- **요청:** 표준 설정 리스트 패턴.

### 반응형
- 세로: [펄스 링]→[BPM/Δ]→[컨트롤]. 가로: 좌측 [펄스 링], 우측 [숫자/안정도/박자/컨트롤]. **둘 다 디자인.**

## 8. 디자이너에게 요청할 산출물 (Deliverables)

- **Figma 파일** (편집 가능, 컴포넌트/오토레이아웃 사용)
- 4개 화면 × (세로·가로) + 공연 모드 시안
- 상태 변형: 펄스 링 초록/노랑/빨강, "측정 중", "신호 없음", "마이크 권한 필요"
- 펄스 링 박동 **애니메이션 사양**(타이밍·이징·크기변화) — 개발자가 Canvas로 구현하므로 수치로 명시
- 디자인 토큰 표(색·타이포·간격) + 컴포넌트(버튼·토글·세그먼트·입력·리스트·미터)
- **아이콘 에셋**: PWA용 192/512px, maskable 512px, apple-touch 180px, favicon SVG (현재 플레이스홀더 교체)
- 앱 이름/로고 제안 (선택)
