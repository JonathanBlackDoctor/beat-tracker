# 비트키퍼 (가칭) — 빌드 브리프 / Build Spec

> **실행 주체:** Claude Code 에이전트
> **목표 산출물:** 안드로이드용 PWA (웹앱). 스토어 등록 없음. 1인 사용(본인 폰).
> **상태:** 모든 설계 결정 완료. 이 문서는 *결정 완료 사양*이다. 새로 정하지 말고 그대로 구현할 것.

---

## 0. 이 문서를 읽는 에이전트에게

- 아래 내용은 사용자와의 사전 협의로 **이미 확정된 사양**이다. "기본값"이라고 쓰인 항목은 사용자가 명시적으로 바꾸기 전까지 최종 기본값이다.
- 모호하거나 서로 모순되는 지점이 **실제로** 막힐 때만 사용자에게 물어라. 그 외에는 이 문서의 결정을 따른다.
- 타깃은 **안드로이드 크롬 최신 버전**. iOS Safari는 v1 대상이 아니다(깨지지만 않게 하되 최적화/검증 불필요).
- 외부 서버·로그인·애널리틱스·결제 **전부 없음**. 모든 동작은 기기 로컬 + 오프라인.

## 1. 제품 개요

밴드에서 퍼커션(카혼/드럼)을 연주하는 주자가 **무대·합주 중** 사용하는 실시간 템포 보조 앱이다.

- 폰 마이크로 **밴드 전체의 템포**를 실시간 추정한다(개별 카혼 소리를 분리하지 않는다 — §6 참조).
- 곡마다 정한 **목표 BPM**과 비교해, 지금 **빨라지는지(앞섬)/느려지는지(처짐)**, 그리고 **얼마나 흔들리는지(안정도)**를 한눈에 보여준다.
- 동시에 **기준 메트로놈**(인이어용 클릭 소리 + 화면 깜빡임)을 제공해 박자를 잡고 유지하게 돕는다.
- 곡들을 **셋리스트**로 저장해두고 공연 중 넘겨가며 쓴다.

**사용 환경 전제:** 무대에서 0.5초 이내로 흘끗 보고 판단. 인이어 모니터 사용. 안드로이드 폰을 거치대에 세로 또는 가로로 거치.

## 2. 핵심 용어

| 용어 | 의미 |
|---|---|
| 목표 BPM (target) | 사용자가 곡/구간에 대해 정한 기준 템포 |
| 감지 BPM (detected) | 마이크 입력으로 실시간 추정한 현재 템포 |
| 템포 편차 (Δ) | `detected − target`. 양수=앞섬(빠름), 음수=처짐(느림) |
| 앞섬/처짐 | Δ의 **부호와 크기**로 표현 (위치/ms 기준 아님 — §6, §10 참조) |
| 안정도 | 최근 구간 동안 감지 BPM의 변동(분산). 낮을수록 안정 |
| 기준 메트로놈 | 목표 템포로 울리는 클릭(소리) + 화면 펄스(시각) |
| 프리 모드 | 목표 없이 감지 BPM·안정도만 표시 |
| 곡 모드 | 목표 BPM 설정 후 비교까지 표시 |
| 셋리스트 | 순서가 있는 곡 묶음 |
| 구간(섹션) | 한 곡 안에서 BPM/박자표가 달라지는 부분 |

## 3. 기술 스택 & 아키텍처

- **형태:** PWA. `Vite + TypeScript` 권장. UI는 가볍게(바닐라 TS 또는 Preact/Svelte 수준). **60fps 시각 렌더링과 오디오 스케줄링은 UI 프레임워크의 리렌더링과 분리**할 것.
- **오디오:** Web Audio API (`AudioContext`). 마이크는 `navigator.mediaDevices.getUserMedia`.
- **클릭 스케줄러:** lookahead 스케줄링("두 개의 시계" 패턴). `setInterval` ~25ms로 깨어나, `AudioContext.currentTime` 기준 ~100ms 앞까지 클릭 이벤트를 미리 예약한다. 샘플 단위로 정확하게. **setInterval 자체로 박자를 치지 말 것**(드리프트 발생).
- **템포 감지 엔진:** §6에 상세. 가능하면 `AudioWorkletProcessor`에서 DSP를 돌려 메인 스레드 부담을 줄인다. (ScriptProcessorNode는 deprecated — 최후 폴백으로만.)
- **렌더링:** 펄스 링/바늘은 Canvas + `requestAnimationFrame`. 숫자·컨트롤은 DOM.
- **저장:** IndexedDB(소량이면 localStorage 허용). 오프라인 우선. **백업/내보내기 기능은 만들지 않는다**(결정).
- **PWA 인프라:**
  - `manifest.json` — name, icons, `display: "standalone"`, `orientation: "any"`, 다크 `theme_color`/`background_color`.
  - Service Worker — 앱 셸 전체 캐싱, 첫 로드 후 **완전 오프라인** 동작.
  - **Wake Lock API** — 연주(클릭/측정) 중 화면이 꺼지지 않게. 정지/백그라운드 시 해제.
  - Vibration API — 진동 피드백 옵션용(단, 폰이 거치대에 있으면 체감 거의 안 됨 — 기본 off).
  - 화면 방향 변화에 반응하는 레이아웃(§4.1).
- **타깃:** 안드로이드 크롬 최신. iOS 비대상.

## 4. 화면 & UX 명세

화면 구성: **(A) 연주 화면**, **(B) 셋리스트 화면**, **(C) 곡 편집 화면**, **(D) 설정 화면**. 연주 화면이 기본 진입점.

### 4.1 연주 화면 (핵심)

시각 요소는 **모두 표시하되 명확한 위계**로 둔다(흘끗 0.5초 판독이 목표):

1. **(메인) 펄스 링** — 화면 중앙의 큰 원. 기준 메트로놈 박자마다 커졌다 작아지며 깜빡인다. **링의 색**이 드리프트 존을 나타낸다(초록/노랑/빨강).
2. **앞섬/처짐 바늘** — 링에 통합. 중앙=목표 일치, **왼쪽=처짐(느림), 오른쪽=앞섬(빠름)**. 위치는 Δ에 매핑(풀스케일 기본 **±8 BPM**). 바늘의 떨림 폭으로 안정도를 함께 표현해도 됨.
3. **(보조) 감지 BPM 숫자(큰 글씨)** + 목표 표기(예: `124 → 목표 120`, `Δ +4`).
4. **안정도 표시** — 작은 막대/미터. 최근 구간 변동.
5. **박자 표시** — 박자표에 맞춘 점(예: 4/4 → 점 4개), 현재 박 강조, 1박 강세.
6. **측정 모드 토글** — `항상 측정` ↔ `탭 측정`. 탭 측정 모드에선 큰 **"측정"** 버튼을 눌러 몇 박을 샘플링.
7. **트랜스포트/컨트롤** — 클릭 시작/정지, 현재 곡명, **"다음 구간"** 버튼(구간이 있는 곡일 때), 이전/다음 곡.
8. **공연 모드(전체화면) 토글** — 모든 컨트롤을 숨기고 1·2·3번을 최대 크기로. 화면 탭하면 컨트롤이 잠깐 나타남.

**반응형:**
- 세로: 위→아래로 [펄스 링] → [BPM 숫자/Δ] → [컨트롤].
- 가로: 좌측 [펄스 링], 우측 [숫자/안정도/박자/컨트롤].
- 두 방향 모두 지원. 방향 전환 시 자동 재배치.

**드리프트 색 임계값(설정 가능, 기본값):** 초록 `|Δ| ≤ 1.5 BPM`, 노랑 `1.5 < |Δ| ≤ 4`, 빨강 `|Δ| > 4`.

**상태 표시:** `마이크 권한 필요`, `측정 중…`(신뢰도 도달 전), `신호 없음 / 박자 불명확`(신뢰도 미달).

### 4.2 셋리스트 화면

- **여러 개의 이름 있는 셋리스트** 지원. 활성 셋리스트 선택.
- 셋리스트 내부: 순서가 있는 곡 목록(곡명·BPM·박자표). 드래그로 순서 변경, 추가/편집/삭제/복제.
- 곡 탭 → 연주 화면에 로드(또는 바로 재생).

### 4.3 곡 편집 화면

- 필드: **곡명**, **기본 목표 BPM**, **박자표**(4/4·3/4·6/8), **메모/큐**(선택).
- **구간(섹션)** — 곡 중간 템포 변화용. 선택적 목록. 각 섹션: 이름(예: 인트로/벌스/후렴), **BPM**, 박자표(미지정 시 곡 기본값 상속), **바 수**(선택, 자동 전환용). 섹션이 없으면 단일 템포 곡.
- **목표 BPM 설정 방법(모두 제공):**
  - 숫자 직접 입력(스테퍼 + 키패드).
  - **탭 템포**(버튼 연타 → 평균 산출).
  - **자동 인식**(연주를 들려주고 "이 템포로 설정" → 몇 박 듣고 스냅).

### 4.4 설정 화면

- 클릭음 종류(예: 우드블록/비프/하이햇/림), 클릭 볼륨, **1박 강세 on/off**, **카운트인**(off / 1마디 / 2마디), **잘게 쪼개기(subdivision)**(off / 8분 / 16분), 드리프트 색 임계값(초록·노랑), **진동 피드백 on/off**(기본 off), **측정 기본 모드**(항상/탭), **화면 항상 켜둠**(기본 on), 언어(한국어).

## 5. 메트로놈 사양

- 기준 클릭은 **현재 활성 목표/섹션 BPM**으로 울린다.
- **박자표별 박 구조:**
  - **4/4** — 한 마디 4박, 1박 강세.
  - **3/4** — 한 마디 3박, 1박 강세.
  - **6/8** — 컴파운드. 8분음표 6펄스/마디, **1박·4박 강세**(점4분음표 2박 느낌). 펄스 링의 "메인 펄스"는 점4분(2박/마디)으로 두고, 8분 펄스는 잘게 쪼개기로 표현 가능.
- **잘게 쪼개기(subdivision)**: 설정 시 오프비트에 약한 클릭 추가.
- **카운트인**: 설정 시 시작 전 1·2마디 미리 클릭(라이브용 기본은 off).
- **시각 깜빡임(펄스)은 오디오 클릭과 동일한 스케줄러 타임스탬프에 정확히 동기**.
- **레이턴시:** 가능하면 `AudioContext.baseLatency`/`outputLatency`를 반영. 클릭은 흔들림 없이 일정해야 함(스케줄러가 권위, setInterval 드리프트 금지).

## 6. 실시간 템포 감지 엔진 (가장 어려운 부분 — 상세)

> 핵심 주의: 기성 BPM 라이브러리는 대부분 **녹음 버퍼 분석용**이라 연속 실시간 추적에 부적합하다. **직접 DSP로 구현**한다(의존성 최소).

**입력 설정**
- `getUserMedia`로 마이크 스트림. **`echoCancellation`, `noiseSuppression`, `autoGainControl`을 모두 OFF**로 요청(원신호 필요. AGC는 온셋 에너지를 왜곡).

**파이프라인**
1. **온셋 검출** — 짧은 FFT 프레임(예: 1024 샘플, hop ~512)에서 **스펙트럴 플럭스**(half-wave rectified) 또는 에너지 기반 온셋 엔벨로프 계산. 타악 온셋에 강함. 가능하면 AudioWorklet에서 수행.
2. **템포 추정** — 최근 4~8초 온셋 엔벨로프 버퍼에 대해 **자기상관(autocorrelation)** 또는 **인터-온셋-인터벌(IOI) 히스토그램**. 유효 범위 **40~240 BPM**에서 피크 → 후보 템포.
3. **옥타브 보정(절반/2배 오인)** — 곡 모드에선 후보를 **목표 BPM에 가장 가까운 옥타브로 스냅**. 프리 모드에선 음악적 범위(예: 70~160) 또는 가장 강한 주기성 선호.
4. **스무딩** — 최근 추정치에 median/EMA 적용해 표시값 안정화. 신뢰도와 분산을 산출(분산 → **안정도**).
5. **편차 계산** — `Δ = detected − target`. 부호 → 앞섬/처짐, 크기 → 바늘 위치·색 존.

**지연/락온 특성(UI에 반영)**
- 템포는 약 **4~8박** 들어야 잡히고, 급변 시 **1~2박** 늦게 반응. 신뢰도 도달 전까지 `측정 중…` 표시.
- 무대 큰 볼륨에서 마이크 클리핑 가능 → 주기성(템포)은 살아남지만 온셋 검출은 저하. 엔벨로프 정규화/클램프. 신뢰도 미달 시 `신호 없음`.

**구현 권장**
- DSP는 `AudioWorkletProcessor`에서(메인 스레드 분리), 온셋 이벤트/엔벨로프만 메인으로 전달. 메인에서 작은 버퍼로 자기상관.
- v1은 **순수 DSP, 외부 ML/대형 라이브러리 없음**. (후일 aubio WASM 등은 선택 사항.)

## 8. 단계별 마일스톤 & 완료 판정

### Phase 1 — 코어 엔진 + 연주 화면 (단일 곡, 수동 BPM)
- 마이크 권한, 실시간 템포 감지 동작, 기준 클릭 + 화면 펄스(목표 BPM), 펄스 링 + 바늘 + BPM 숫자 + 색 존, **항상 측정/탭 측정 토글**, Wake Lock, 세로·가로 반응형, 다크 테마, 공연 모드(전체화면).
- **완료 판정:** 폰 근처에서 손/메트로놈으로 알려진 BPM을 치면 약 8박 내에 ±2 BPM로 표시. 바늘/색이 Δ를 반영. 클릭이 5분간 흔들림 없이 유지.

### Phase 2 — 셋리스트·곡 관리 + 박자표 + 구간(곡 중간 템포 변화 v1) + 설정
- 여러 셋리스트, 곡 편집(섹션 포함), **"다음 구간" 수동 전환**, 모든 BPM 설정법(숫자/탭/자동), 카운트인, 잘게 쪼개기, 클릭음·볼륨, 임계값 설정.
- **완료 판정:** 3곡 셋리스트 + 2개 섹션 곡을 만들어, 공연 중 곡 전환·구간 전환이 동작. 새로고침(오프라인) 후에도 설정/데이터 유지.

### Phase 3 — 기록·리뷰 (부가 기능, 우선순위 최하)
- 곡 재생 중 감지 BPM·Δ를 시간축으로 로깅. 정지 후 **곡별 타임라인/그래프**로 "언제 얼마나 흔들렸는지" 표시(목표선 포함). 최근 세션 로컬 저장.
- **완료 판정:** 한 곡 후 템포-시간 그래프(목표선)를 확인. 새로고침 후에도 유지.
- (참고: 이 기능은 "있으면 좋지만 부가" 결정 → 가장 마지막에.)

## 9. 비기능 요건

- **오프라인:** 첫 로드 후 완전 오프라인(서비스 워커가 모든 자산 캐시).
- **설치형:** manifest + 홈 화면에 추가 안내.
- **화면 유지:** 연주 중 Wake Lock, 정지/백그라운드 시 해제.
- **흘끗 판독성:** 팔 거리에서 핵심 상태가 0.5초 내 인지. 크고 고대비.
- **성능:** 60fps 시각, 오디오 스케줄링 지터 수 ms 이내, 연주 중 GC 멈춤 없게(버퍼 사전할당, 프레임마다 할당 회피).
- **프라이버시:** 마이크 오디오는 **기기를 떠나지 않음**. 원음 저장 없음(Phase 3에서도 파생된 BPM 시계열만 저장).
- **언어:** 한국어 UI.
- **엣지 케이스 처리:** 권한 거부, 신호 없음, 과대 입력(클리핑), 백그라운드 전환(클릭 일시정지 + Wake Lock 해제, 복귀 시 깔끔히 재개).

## 10. 가정 & 메모 (에이전트 참고)

- **구간 전환은 수동**("다음 구간" 버튼)이 기본. 바 수 기반 자동 전환은 선택 설정.
- **앱 이름 미정** — 작업명 "비트키퍼". 에이전트가 더 나은 이름을 제안해도 좋음.
- **iOS는 v1 비대상.**
- **앞섬/처짐은 템포 기준만**(빠름/느림). 클릭 대비 한 박 한 박의 위치(ms) 피드백은 **밴드 합주 마이크로는 부정확**하여 범위 외. 만약 추후 그게 꼭 필요하면 **카혼에 콘택트(피에조) 마이크**를 다는 별도 설계가 필요(현재 결정: 템포 기준 only).
- 아이콘/스플래시: 다크 테마 기반 간단한 플레이스홀더 생성.
